Transition : Pour récapituler, quelques points vraiment importants
====

// ─── DO / DON'T ──────────────────────────────────────────────────────────────

[%notitle]
== Ce qu'on fait

Ce qu'on fait :

[%step]
* Définir un schéma de log *dès le départ*
* ILM et rétention *dès J1*
* Une alerte = un runbook (même minimal)
* Mesurer le MTTR, pas le nombre de métriques
* Surveiller le système de monitoring lui-même

[NOTE.speaker]
====
Pour commencer, ce qu'on fait

* Ces points reviennent dans les 3 REXs sous des formes différentes
* Step : Le schéma de log : fondation de tout — sans ça, impossible de corréler les 3 REXs ensemble
* Step : ILM dès J1 : le remédier plus tard coûte 10x plus cher
* Step : Une alerte = un runbook (même minimal)
* Step : On mesure le MTTR — Mean Time To Repair — c'est la seule métrique que le business comprend
* Step : Surtout on surveille le système de monitoring lui-même

Transition : ce qu'on evite
====

[%notitle]
== Ce qu'on évite

Ce qu'on évite :

[%step]
* Les logs en mode DEBUG en production
* Alerter sur tout — c'est alerter sur rien
* Faire confiance aux estimations de volumétrie — mesurez
* Monitorer sans inventaire — vous ne savez pas ce que vous ne monitorez pas

[NOTE.speaker]
====
* Step : On evite le DEBUG en prod : 60 Go/jour pour un seul outil. Vu dans le REX 1.
* Step : On évite d'alerter sur tout : les équipes ops ont coupé les notifications dans le REX 3. C'est l'échec du monitoring.
* Step : On évite de faire confiance aux Estimations de volumétrie : toujours fausses => On mesure. Dans le REX 2, on est passé de "quelques Go" à 782 Go réels.
* Step : On évite de monitorer sans inventaire : vous ne savez pas ce que vous monitorer

== La checklist lessive

[cols="^.^1,1",frame=none,grid=none]
|===
a|
image::synthesis-checklist.svg[checklist lessive]
a|
[%step]
* 👕 *Trier* +
Parsing, métadonnées, schéma de log
* 🧺 *Dégraisser* +
Nettoyage, rétention, ILM
* 🏭 *Automatiser* +
Alerting intelligent, runbooks, labels
|===


[NOTE.speaker]
====
* Step : Trier : c'est le REX 1. Séparer le coton du synthétique. Un log non étiqueté est une chaussette orpheline.
* Step : Dégraisser : c'est le REX 2. La meilleure donnée, c'est celle qu'on n'a pas besoin de stocker.
* Step : Automatiser : c'est le REX 3. L'alerte doit dire "voici ce que tu dois faire", pas "il y a un problème".

Ces 3 étapes sont dans cet ordre. On ne peut pas automatiser ce qu'on n'a pas trié. On ne peut pas dégraisser ce qu'on n'a pas catégorisé.
