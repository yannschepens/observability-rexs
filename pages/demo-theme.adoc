== Slide standard

Le chrome (header, bande rouge, footer) s'applique *automatiquement* sur toutes les sections.

* Pas de classe à ajouter
* Pas de HTML dans le `.adoc`
* Logo miniature dans le header via `::before`
* Tagline dans le footer via `::after`

[NOTE.speaker]
====
Le sélecteur .reveal.slide porte le background multi-couches.
Rien à faire dans l'asciidoc pour avoir le chrome.
====

== Blocs de code

Les blocs `source` sont automatiquement stylisés : fond marine, texte bleu clair, bordure rouge.

[source,text]
----
ERROR 2024-01-12 ▒▒▓ NUL unexpected token
WARN  malformed timestamp: -9999-99-99
----

Inline : `{"level":"error","code":401}` — même style en inline.

[source,json]
----
{"level": "info", "ts": "2024-01-12T14:32:01Z",
 "msg": "pipeline clean", "svc": "nl-engine"}
----

[.nl-dark]
== Composants inline

Classes `.nl-badge` et `.nl-log-lines`

== Badges rétro

Utilisation inline avec la syntaxe de rôle AsciiDoc `[.classe]#texte#` :

[.nl-badge]#ISO LOG-9001# [.nl-badge]#Niveau 3 PROPRETÉ# [.nl-badge]#CLEAN METRICS#

{nbsp}

[.nl-badge.founded]#Fondée en 1987# [.nl-badge.founded]#SINCE 87# [.nl-badge.founded]#EST. 1987#

[NOTE.speaker]
====
[.nl-badge]#texte# → double bordure marine, fond crème, Courier New.
[.nl-badge.founded]#texte# → variante rouge.
Utilisable en inline directement dans une phrase.
====
